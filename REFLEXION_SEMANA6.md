# Reflexión Semana 6

## 25. ¿Por qué Git ayuda a reducir pérdida de trabajo?

Git almacena un historial de cambios que permite recuperar versiones anteriores del proyecto y evitar la pérdida de información.

## 26. ¿Por qué GitHub aporta al trabajo colaborativo?

Permite que varias personas trabajen sobre el mismo proyecto, compartan cambios y mantengan sincronizado el código.

## 27. ¿Qué ventaja tiene trabajar con ramas?

Las ramas permiten desarrollar nuevas funcionalidades sin afectar la versión principal del proyecto.

## 28. ¿Cómo se relaciona Git con integración continua o despliegue continuo?

Git proporciona el control de versiones necesario para que herramientas de CI/CD automaticen validaciones, pruebas y despliegues.

## 29. ¿Qué deuda técnica puede evitarse usando control de versiones?

Se evita la pérdida de cambios, duplicación de trabajo, conflictos de versiones y falta de trazabilidad.
