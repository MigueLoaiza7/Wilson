# Reflexión Aplicada - Semana 7

## 37. ¿Qué es un workflow?

Un workflow es un proceso automatizado definido en un archivo YAML que permite ejecutar tareas dentro de GitHub Actions cuando ocurre un evento específico, como un push o una ejecución manual.

## 38. Diferencia entre job y step

Un job es un conjunto de tareas que se ejecutan en un mismo runner. Un step es una acción individual dentro de un job, como ejecutar un comando o utilizar una acción de GitHub.

## 39. ¿Para qué sirve un runner?

Un runner es la máquina o entorno donde GitHub Actions ejecuta los jobs definidos en un workflow.

## 40. ¿Cuál es una ventaja de usar artifacts?

Los artifacts permiten almacenar y descargar archivos generados durante la ejecución de un workflow, facilitando la conservación de evidencias y resultados.

## 41. ¿Cómo GitHub Actions ayuda a disminuir errores manuales?

GitHub Actions automatiza tareas repetitivas como validaciones, pruebas y despliegues, reduciendo la intervención manual y disminuyendo la posibilidad de errores humanos.
